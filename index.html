The project is based on a class called ItemTracker. The ItemTracker class allows users to search for items, display item frequency and print them into another file, display item frequency in the form of a histogram and print it into another file, change the input file containing the items to another file, and change the name of the output file containing the results. This class is designed to work with files only, but may be expanded in further updates to increase flexibility. The diagrams should be in the "Documentation" folder.

What did you do particularly well on?

I did particularly well when it came to designing and implementing the class and the class member methods/functions. I tried to keep the file streaming logic to only have the scope of the method to avoid any leaks. I thought I did well with the standard template library maps, but I think I will spend some time practicing and studying it for future uses.

What skills from this project will be particularly transferable to other projects?

I believe I could enhance the code to process multiple files at the same time using multi-threading or simply have each file linearly processed to increase efficiency. I could also add an option to change the file search using an item name to be case-sensitive, ignore capitalizations and spaces, and possibly process binary, decimal, and hexadecimal encoding to enhance flexibility and functionality. Most of the class’s methods/functions could have exception handling to avoid input errors and file streaming errors to enhance security. I’ve thought about making the code work with networks to increase flexibility and functionality, but I would need to address the security part of that. But I think the top priority will need to be assessing the projects’ pros and cons and improving the code itself to be more efficient and understandable. There are so many ways I could change how the software project works.
